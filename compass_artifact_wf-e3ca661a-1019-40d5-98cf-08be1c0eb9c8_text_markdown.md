# Интеграция ИИ в Telegram Mini App для изучения английского при нулевом бюджете

**Для проекта Wordy доступен работоспособный путь к полноценной ИИ-интеграции без единого рубля затрат на старте.** Комбинация из 2–3 бесплатных API-провайдеров обеспечит ~80–100 запросов в день, а агрессивное прекэширование контента для всех 2 800 слов обойдётся в $3–5 разовых затрат (токенами бесплатных тиров). При этом **90% ИИ-фич можно реализовать без real-time вызовов к API вообще** — сгенерировав контент заранее и сохранив в PostgreSQL. Это ключевой архитектурный принцип: с базой из 2 800 слов полное прекэширование примеров, мнемоник, объяснений и подсказок занимает ~22 МБ в БД и устраняет зависимость от внешних API для большинства пользовательских взаимодействий.

При покупке кредитов на $10 лимит вырастает до **1 000 запросов в день**. Также стоит исследовать российские агрегаторы **VseLLM.ru** и **Polza.ai**, предоставляющие API-доступ к западным моделям с оплатой в рублях.

## Десять ИИ-фич и какие из них можно прекэшировать

Критически важное разделение: фичи, которые можно сгенерировать заранее для всех 2 800 слов и сохранить в БД (нулевая стоимость на каждого нового пользователя), и фичи, требующие real-time обращений к API (стоимость растёт с каждым пользователем).

### Прекэшируемые фичи (генерация один раз → хранение навсегда)

**Примеры предложений со словом** — три предложения разной сложности (A1, B1, C1) для каждого слова. Разовая стоимость генерации: ~560K выходных токенов, **~$0.34** на GPT-4o mini. Это must-have для MVP — каждое приложение для изучения лексики должно показывать слово в контексте. Предложения не зависят от пользователя и могут пройти ручную проверку перед деплоем.

**Мнемоники для запоминания** — ассоциации, связывающие английское звучание с русскими словами. Например: «ABANDON звучит как 'а бандон' — бандит бросает (abandons) место преступления». Стоимость: ~700K токенов, **~$0.42**. Уникальный selling point для русскоговорящей аудитории — ни Duolingo, ни Lingvist не предлагают мнемоники с русскими ассоциациями.

**Объяснения грамматических правил** — ~50–100 тем (артикли, времена, предлоги), которых нет в русском. Стоимость: ~50K токенов, **~$0.03**. Привязываются к словам в базе.

**Парафразирование** — 3–5 синонимов и альтернативных способов выразить то же значение (formal, casual, slang). Стоимость: ~560K токенов, **~$0.34**.

**Адаптивные подсказки** — лестница из 3–4 хинтов для каждого слова (категория → первая буква → предложение с пропуском). Стоимость: ~420K токенов, **~$0.25**. Снижает фрустрацию в квизах и PvP-дуэлях.

### Гибридные фичи (прекэш + точечный real-time)

**Объяснение ошибок в квизах** — 80% ошибок предсказуемы (типичные путаницы: affect/effect, its/it's). Для каждого слова заранее генерируются объяснения частых ошибок. Оставшиеся 20% — follow-up вопросы пользователя — обрабатываются через API в реальном времени. Это аналог фичи **Explain My Answer** в Duolingo Max, которая стала главным драйвером конверсии в платную подписку.

**Персонализированные квизы** — пул из 5 вариаций вопросов на каждое слово генерируется заранее (~2.1M токенов, ~$1.26). Выбор вопросов под слабые места пользователя — алгоритмический (SRS в Wordy уже это делает), LLM не нужен.

**Генерация историй** — шаблонные истории по темам прекэшируются; персонализированные (с конкретными словами пользователя) генерируются на лету (~500 токенов за историю, **$0.0005**).

### Только real-time (нельзя прекэшировать)

**AI-чатбот для разговорной практики** — ролевые диалоги (заказ в ресторане, собеседование). **2 000–5 000 токенов за сессию**, ~$0.001–0.003 на GPT-4o mini. Требует WebSocket/SSE, модерации, управления контекстом. Самая сложная в реализации фича — Duolingo потратил месяцы на её разработку с OpenAI. Рекомендация: **отложить на Phase 2–3**.

**Проверка написанного текста** — пользователь пишет предложение, ИИ корректирует. ~500–1 500 токенов за проверку. Инфраструктурно пересекается с чатботом — реализовывать вместе.

### Как это делают лидеры рынка

**Duolingo Max** использует GPT-4 для двух фич: Explain My Answer (объяснение ошибок в чате) и Roleplay (свободный диалог). Результат: **+78% готовности к реальным разговорам**, +20–30% ретеншена у когорт с ИИ. Генеративный ИИ генерирует «close to 100%» обучающего контента, ускорив создание курсов в 10 раз. Цена Max: $29.99/мес.

**Lingvist** — ближайший аналог Wordy (фокус на лексику, ~3 000 слов, SRS). ИИ в Lingvist *аналитический*, а не *генеративный*: оптимизирует интервалы повторения, отслеживает паттерны ошибок. Генеративного контента нет. Это значит, что **добавление генеративных фич в Wordy даст конкурентное преимущество** даже над Lingvist.

## Архитектура: как выжать максимум из минимальных ресурсов

### Схема данных для прекэшированного ИИ-контента

```prisma
model WordAiContent {
  id            Int      @id @default(autoincrement())
  wordId        Int
  word          Word     @relation(fields: [wordId], references: [id])
  contentType   String   // 'sentences' | 'mnemonic' | 'grammar' | 'paraphrases' | 'hints'
  content       Json     // Структурированный JSON
  model         String   // 'gemini-2.5-flash' | 'gigachat-2'
  promptVersion Int      @default(1)
  tokensCost    Int
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  @@unique([wordId, contentType, promptVersion])
  @@index([wordId, contentType])
}

model AiUsageLog {
  id        Int      @id @default(autoincrement())
  userId    Int
  endpoint  String
  tokensIn  Int
  tokensOut Int
  provider  String
  cached    Boolean  @default(false)
  createdAt DateTime @default(now())
  @@index([userId, createdAt])
}
```

Объём данных для 2 800 слов × 5 типов контента: **~22 МБ** в PostgreSQL. На 40 ГБ диске это незаметно.

### Батч-генерация ночным кроном

При бесплатном лимите Gemini Flash (~500 RPD) генерация всех 2 800 слов × 4 типа контента = **11 200 запросов** займёт **~45 ночей**. Скрипт запускается в 2:00, обрабатывает ~250 слов, делает паузу 7 секунд между запросами (укладываясь в 10 RPM). При ротации между 2–3 провайдерами срок сокращается до **15–20 ночей**. Приоритет: слова по частотности — сначала 500 самых распространённых.

### Абстракция над провайдерами (легко менять API)

```typescript
// src/ai/types.ts
export interface ILLMProvider {
  readonly name: string;
  complete(opts: {
    messages: AiMessage[];
    temperature?: number;
    maxTokens?: number;
    responseFormat?: 'json' | 'text';
  }): Promise<{ content: string; model: string; tokensIn: number; tokensOut: number }>;
  stream(opts: AiCompletionOptions): AsyncGenerator<AiStreamChunk>;
}

// src/ai/index.ts — фабрика
export function createProvider(): ILLMProvider {
  switch (process.env.AI_PROVIDER) {
    case 'gemini':   return new GeminiProvider(process.env.GEMINI_API_KEY!);
    case 'gigachat': return new GigaChatProvider(process.env.GIGACHAT_AUTH!);
    case 'groq':     return new GroqProvider(process.env.GROQ_API_KEY!);
    case 'openai':   return new OpenAIProvider(process.env.OPENAI_API_KEY!);
    default:         return new GeminiProvider(process.env.GEMINI_API_KEY!);
  }
}
```

Все провайдеры реализуют один интерфейс `ILLMProvider`. Смена провайдера — одна переменная окружения. Все поддерживают OpenAI-совместимый формат API (Groq, OpenRouter, Gemini через adapter), что упрощает реализацию.

### Трёхуровневый fallback: кэш → БД → API

```typescript
export async function getAiContent(wordId: number, type: string) {
  // Уровень 1: In-memory кэш (node-cache, ~5 МБ RAM)
  const cached = memoryCache.get(`ai:${wordId}:${type}`);
  if (cached) return { data: cached, source: 'memory' };

  // Уровень 2: PostgreSQL (прекэшированный контент)
  const db = await prisma.wordAiContent.findFirst({
    where: { wordId, contentType: type },
    orderBy: { promptVersion: 'desc' }
  });
  if (db) {
    memoryCache.set(`ai:${wordId}:${type}`, db.content, 86400);
    return { data: db.content, source: 'db' };
  }

  // Уровень 3: Real-time API (с circuit breaker)
  if (circuitBreaker.isOpen()) {
    return { data: null, source: 'unavailable', message: 'ИИ временно недоступен' };
  }
  try {
    const result = await aiProvider.complete(buildPrompt(type, wordId));
    // Сохраняем для будущих запросов
    await prisma.wordAiContent.create({ data: { wordId, contentType: type, ... } });
    return { data: result, source: 'realtime' };
  } catch {
    circuitBreaker.recordFailure();
    return { data: null, source: 'queued', message: 'Генерируем. Попробуйте через минуту.' };
  }
}
```

**Redis не нужен** — при 4 ГБ RAM и масштабе до 1 000 пользователей достаточно `node-cache` в оперативной памяти (~5–10 МБ).

### Rate limiting по Telegram user ID

```typescript
const freeAiLimiter = rateLimit({
  windowMs: 24 * 60 * 60 * 1000, // 24 часа
  limit: 20,                      // 20 AI-запросов/день для free
  keyGenerator: (req) => `user_${req.telegramUser?.id}`,
  message: { error: 'Лимит ИИ исчерпан. Оформите Premium для неограниченного доступа.' }
});

const premiumAiLimiter = rateLimit({
  windowMs: 24 * 60 * 60 * 1000,
  limit: 100,                     // 100 AI-запросов/день для Premium
  keyGenerator: (req) => `user_${req.telegramUser?.id}`,
  skip: (req) => !req.user?.isPremium
});
```

Ключ — **Telegram user ID**, не IP (пользователи могут сидеть за одним IP через NAT). Free-пользователи получают 20 запросов/день — хватит на ~2 квиз-сессии с объяснениями. Premium — 100 запросов/день, включая чат.

## Практические промпты, минимизирующие расход токенов

### System prompt (~35 токенов)
```
You are an English tutor for Russian speakers. Be concise. Use simple explanations.
Include Russian translations in parentheses. Reply in JSON only.
```

### Генерация 3 примеров предложений (~60 вход / ~120 выход)
```
Word: "{word}" [{pos}]
Translation: "{ru}"
Generate 3 sentences: easy(A1), medium(B1), hard(C1).
JSON: {"sentences":[{"level":"A1","en":"...","ru":"..."},{"level":"B1","en":"...","ru":"..."},{"level":"C1","en":"...","ru":"..."}]}
```

### Объяснение грамматической ошибки (~70 / ~100)
```
Wrong: "{wrongAnswer}"
Correct: "{correctAnswer}"
Question: "{questionText}"
Explain error in 2-3 sentences. Include rule name.
JSON: {"rule":"...","explanation":"...","tip":"...","ru":"..."}
```

### Генерация мнемоники (~45 / ~70)
```
Word: "{word}" → "{ru}"
Create memorable association using Russian sound similarity. Max 2 sentences.
JSON: {"mnemonic":"...","association":"...","image":"..."}
```

Пример выхода: `{"mnemonic":"ABANDON sounds like 'а бандон' — бандит abandons the scene","association":"бандит → abandon","image":"🏃‍♂️💨"}`

### Ролевой диалог (~80 / ~100)
```
Scenario: {scenario}
Level: {level}
Start dialogue (3-4 exchanges). Student=customer. Use words: [{wordList}].
JSON: {"scenario":"...","opening":"...","hints":["..."],"vocabulary":["..."]}
```

### Адаптивная подсказка (~55 / ~40)
```
Word: "{word}" [{pos}]
Attempts: {n}, last wrong: "{wrong}"
Hint #{level}: 1=category, 2=first letter+context, 3=sentence with blank.
JSON: {"hint":"...","hintType":"..."}
```

**Главные приёмы экономии токенов**: использовать `responseMimeType: 'application/json'` (убирает «болтовню» модели); убрать слова вежливости из промптов; явно ограничивать длину выхода («Max 2 sentences»); ставить `maxTokens` параметром API; для батч-генерации отправлять **5 слов за один запрос**, амортизируя system prompt.

## Стриминг ответов в Telegram Mini App работает

SSE (Server-Sent Events) **полностью поддерживается** во всех клиентах Telegram: Android (Chromium 119+ WebView), iOS (WKWebView), Desktop (Embedded Chromium). Это стандартный подход, который используют OpenAI и Anthropic для стриминга. Для Wordy рекомендуется SSE вместо WebSocket: проще, однонаправленный (идеально для вывода LLM), автоматически переподключается, работает через все прокси.

На бэкенде — Express-эндпоинт с заголовком `Content-Type: text/event-stream` и обязательным `X-Accel-Buffering: no` для Nginx. На фронтенде — React-хук `useAiStream` с `fetch` + `ReadableStream`. UX при ожидании 2–5 секунд: моментально показать typing dots (три пульсирующие точки), затем посимвольное появление текста с курсором-блинкером, haptic feedback через `window.Telegram.WebApp.HapticFeedback.impactOccurred('light')` при получении первого токена. Кнопка отмены — через `AbortController`.

**Критически важно** для Nginx:
```nginx
location /api/ai/stream {
    proxy_pass http://localhost:3000;
    proxy_buffering off;
    proxy_cache off;
    proxy_http_version 1.1;
    proxy_set_header Connection '';
    proxy_read_timeout 300s;
}
```

## Почему не стоит запускать LLM локально на VPS с 4 ГБ RAM

На VPS с 4 ГБ RAM, где уже работают Node.js, PostgreSQL и другие проекты, свободно остаётся **~2–2.5 ГБ**. Этого хватит только на модели уровня **Qwen 2.5 0.5B** (350 МБ модель + 500 МБ overhead ≈ 1 ГБ) — но качество 0.5B-моделей для language learning задач **неприемлемо**: частые галлюцинации, неспособность объяснить грамматику, примитивные предложения.

| Модель | Размер (Q4) | Нужно RAM | Скорость на CPU | Качество |
|---|---|---|---|---|
| Qwen 2.5 0.5B | ~350 МБ | ~1 ГБ | 15–25 tok/sec | Очень низкое |
| TinyLlama 1.1B | ~700 МБ | ~2 ГБ | 8–15 tok/sec | Низкое |
| Phi-3 Mini 3.8B | ~2.3 ГБ | ~4.5–5.5 ГБ | 2–5 tok/sec | Первый приемлемый |
| Llama 3.1 8B | ~4.5 ГБ | ~8 ГБ | 1–3 tok/sec | Хорошее |

**Phi-3 Mini** — первая модель с приемлемым качеством для обучения языку — **не влезает** в 4 ГБ (нужно 5.5 ГБ только для неё). Вердикт: **используйте бесплатные API-тиры вместо локального запуска**. Если хотите локальную модель — апгрейд VPS до 8 ГБ RAM (~$10–15/мес. на Timeweb/Selectel) и запуск Phi-3 Mini или Qwen 2.5 3B.

## Поэтапная миграция от нуля к масштабу

### Phase 1: Запуск без бюджета (0 ₽)

Прекэшировать контент для всех 2 800 слов через бесплатные API за 15–45 ночей. Реализовать 5 прекэшируемых фич (примеры, мнемоники, грамматика, подсказки, парафразы). Минимальный real-time — объяснение ошибок в квизах через GigaChat + OpenRouter. Лимит для free-пользователей: **5 AI-запросов/день** (тизер Premium). Ёмкость: **50–100 активных пользователей**.

### Phase 2: Первая выручка ($10–50/мес.)

Лучшая стоимость за запрос — **Groq Llama 3.1 8B** ($0.034 за 1 000 запросов), но нужна оплата в USD. Решение: **OpenRouter с криптовалютой** (USDC) — обходит санкционные ограничения, лимит вырастает до 1 000 запр./день, доступ к десяткам моделей через единый API. Альтернатива на рублях: **GigaChat 2 Lite** — пакет 20M токенов за **1 300 ₽** (~40 000 запросов). Разблокировать AI-чатбот для Premium (5 диалогов/день). Ёмкость: **200–500 активных пользователей**.

| Провайдер | Стоимость 1 000 запросов | 100 пользователей/мес. | 1 000 пользователей/мес. | Оплата из РФ |
|---|---|---|---|---|
| Groq Llama 3.1 8B | $0.034 | $2 | $20 | Крипто через OpenRouter |
| Gemini Flash-Lite | $0.14 | $8 | $84 | Google Cloud (неясно) |
| GigaChat 2 Lite | ~32.5 ₽ ($0.35) | ~1 950 ₽ | ~19 500 ₽ | ✅ Рубли, карта РФ |
| Mistral Small | $0.22 | $13 | $132 | ЕС-компания, проще |
| YandexGPT Lite | $0.83 | $50 | $500 | ✅ Рубли |

Расчёт исходит из **20 запросов/день на пользователя** (600 запросов/мес.), ~200 вх. + ~300 вых. токенов на запрос.

### Phase 3: Рост ($100–500/мес.)

Мультипровайдерная маршрутизация: простые запросы (поиск слова, базовая грамматика) → Groq 8B (дешевле всего); средние (коррекция, объяснения) → Gemini Flash; сложные (свободный чат, эссе) → Gemini Pro или Claude Haiku. A/B тестирование: кнопки 👍/👎 на каждом ИИ-ответе, отслеживание quality score по провайдерам. Batch API (Gemini, Claude предлагают **50% скидку** для неспешных запросов) — использовать для ночной регенерации контента при обновлении промптов. Ёмкость: **1 000–5 000 активных пользователей**.

### Решение проблемы оплаты из России

- **GigaChat и YandexGPT** — оплата в рублях, карты РФ, никаких ограничений
- **OpenRouter** — принимает **криптовалюту (USDC)**, обходит санкционные ограничения на USD-платежи
- **Mistral** — французская компания (ЕС), менее жёсткая санкционная политика

## Технические ловушки, которые нужно предвидеть

### Галлюцинации LLM в контексте обучения языку — главный риск

LLM может уверенно выдать **неправильное грамматическое правило**, несуществующий фразовый глагол или ошибочный перевод. Исследования подтверждают, что галлюцинации — математически неизбежное свойство языковых моделей. В контексте обучения это особенно опасно: ученик не сможет отличить правильное объяснение от ложного.

Стратегии защиты: поддерживать **верифицированную базу** словарных данных (Oxford, Cambridge) и использовать RAG — подавать правила и переводы из базы как контекст, а LLM использовать только для генерации user-friendly объяснений. Для критического контента (переводы, грамматические правила) применять **dual-model verification** — запрашивать две разные модели и показывать ответ только при совпадении. Всегда **temperature 0.1–0.3** для фактических ответов, 0.7 только для творческих задач (мнемоники, истории). Добавить кнопку «Ошибка в ответе ИИ» на каждый сгенерированный блок.

### Модерация контента в AI-чате — дёшево и эффективно

Пользователи будут пытаться «взломать» чатбот или использовать его не по назначению. Бюджетные решения: **system prompt hardening** (жёсткие инструкции оставаться в роли учителя английского); **OpenAI Omni-Moderation API** — **бесплатный** инструмент для фильтрации вход/выхода; **Llama Guard 4** через Groq ($0.20/1M токенов) — классификатор безопасности контента; ограничение длины сообщений (500 символов) и **topic detection** перед отправкой в LLM.

### Приватность и персональные данные

**Никогда не отправлять** Telegram user ID, имена или профильные данные в LLM API. Использовать анонимные session ID. Для пользователей из РФ: GigaChat и YandexGPT хранят данные в России (упрощает соответствие 152-ФЗ). Бесплатный тир Gemini — Google **может использовать данные для улучшения продуктов**; платный тир — enterprise-grade privacy. Указать в описании бота, что сообщения обрабатываются ИИ и какими провайдерами.

### Ограничения Telegram Mini App WebView

JS-бандл должен быть **менее 650 КБ** (рекомендация Telegram Store). Time to First Paint — менее 800 мс на 3G. Во время стриминга ограничить обновления DOM до **10–15 раз/сек**, группировать чанки. На iOS — баг с клавиатурой (контент сдвигается при открытии). Нет доступа к микрофону/камере из Mini App. Нет гарантии персистентности localStorage между сессиями — критические данные хранить только на бэкенде.

## Что реализовать в первые четыре недели

Roadmap для Phase 1 MVP — **пять прекэшируемых фич**, которые стоят $0 в эксплуатации:

1. **Примеры предложений** (3 уровня сложности) — прекэш за ~$0.34
2. **Объяснения грамматических правил** — прекэш за ~$0.03
3. **Объяснение ошибок в квизах** — прекэш частых ошибок ($0.50) + real-time для follow-up
4. **Адаптивные подсказки** (лестница хинтов) — прекэш за ~$0.25
5. **Мнемоники с русскими ассоциациями** — прекэш за ~$0.42

**Суммарная стоимость генерации**: ~$1.54 токенами (покрывается бесплатными тирами за 15–45 ночей). **Суммарная стоимость эксплуатации**: $0/мес. для любого количества пользователей. Эти пять фич покрывают **80% ценности ИИ-интеграции** при нулевых операционных затратах и превращают Wordy из простого квиз-приложения в интеллектуального репетитора английского — конкурентный уровень Duolingo Max, но при стоимости на порядки ниже.

Ключевая мысль: **2 800 слов — это не слабость, а преимущество**. База достаточно мала, чтобы прекэшировать абсолютно всё, и достаточно велика, чтобы покрыть ~85% повседневного английского. Прекэширование трансформирует ИИ из переменной статьи расходов в фиксированную однократную инвестицию.
